>First off, GIF files are binary.  Not MacBinary, but binary.  When you FTP
>from gaffa.mit.edu (at least 'til our new pseudo-moderator gets set up),
>make sure you set binary on FTP (by typing "binary [CR]".  When you download
>to your mac, again, you need to use binary, but not MacBinary.

	I'd like to make a point here if I may. I, too, have done some FTPing
from gaffa.mit.edu. My procedure is this. First of all I have to log on to
a mini computer on campus. I do this from home using my Mac II (actually it's
my boss's). I run FTP and down load stuff on to the campus machine. From here 
I use ZMODEM (which I use to log on in the first place) in YMODEM mode (this 
is over a 1200/75 baud so it's not very quick) to download th gifs onto my 
Mac. Lo and Behold I use (wait for it!).....

		Smart Macbinary.
		^^^^^^^^^^^^^^^^
	I would assume this is slightly different to Macbinary but I haven't
tried straight Macbinary. I'm not sure if this is a feature of ZModem (which
I'm using even as I type) or not. I would assume so as  I haven't seen it 
elsewhere. Therefore a piece of advice to Tom is to use ZModem as I haven't
experienced any problems whatsoever.
