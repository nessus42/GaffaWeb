Sigh.  I've been trying to do this unsuccessfully for a week now.  We're
having problems with the anonymous ftp server here at Apple, and I've
been having troubles with my remote access to the campus network.  I
finally got my end worked out yesterday, I think, but there's another
problem on that end.  I mention all this so you know I've *really* been
trying to make these things available!

I've done the conversion from .wav to .snd, but it was such a brute force
hack job that I doubt many people will take the effort.  The three main
tricks are, use ResEdit to change the file type and creator to match
some sound program you have (such as SoundEdit Pro), use that program
to read the file and cut off the first few samples (they aren't really
samples, but some header information that the .wav file contained, I
believe) and then save the file as a System 7 "snd" file (if your
software lets you set the sample rate - *without* resampling the data -
do so, and you can skip the next step; mine wouldn't let me), then go
back to ResEdit and replace the hexadecimal sample rate (my software
defaulted to 22kHz, it needs to be 11kHz; to save you some trouble,
the 29th through 32nd bytes, numbering from 1, of the "snd " resource
should be 2B77 45D0).

As horrible as that procedure sounds, it doesn't touch the actual
samples, so there's no loss in quality, and it leaves things such that
the "snd" file can be played from the Finder with no other applications.

Or, I hope to make the files available within a couple of days.
