   This is going out to most music groups.  There was a lyrics server
at UMass that had a lot of lyrics, and you were able to send lyrics to it 
as well as get copies.  It is currently not active, but if someone is 
willing and knowledgeable to port it successfully, all of the net would be 
grateful (well, at least me!).

  Here is a summary of how the lyrics server works. Note that it is 
CURRENTLY NOT WORKING:

***************

>>[Extracted from the Gibralter Digest ]
>>
>>III.  Lyrics Server Instructions Summary
>> 
>>    These are excerpts from the help files which can be obtained from the
>>lyrics server at UMass.  You can get the whole file by sending the command
>>'help' to the server.  I would like to point out that all lyrics are user
>>contributed, and that the classic rock genre is sparsely represented.  This,
>>of course, can be rectified by our readers (you might take that as a hint).
>> 
>>Disclaimer:  All actions taken by the server as response to your
>>requests are your own responsibility.
>> 
>>The format for a request from the lyrics server is as follows:
>>Command/param1=option/param2=option/param3=option
>> 
>>There are 3 commands available for requests.
>>   Lyrics - prints lyrics
>>   Albums - prints album titles and authors
>>   Songs  - prints album titles, authors, and songs on albums
>> 
>>These commands will accept 3 parameters.
>>   /Author=xx   restricts command to response that is the work of author xx
>>   /Album=xx    restricts command to responses contained on album xx
>>   /Song=xx     restricts command to responses of song titled xx
>> 
>>Multiple parameters are also valid.
>> 
>>Commands and parameters may be abbreviated to their minimum uniqueness, thus
>> 
>>Albums/Author=Peter Gabriel/Song=Red Rain
>>     is equivalent to
>>Al/Au=Peter Gabriel/s=Red Rain
>> 
>>The lyrics server wildcard string search will seek out character
>>sequences in the three valid fields of Author, Album, and Song.
>>The wildcard character is a +, and there may be any number of +'s in
>>the string in any position.
>> 
>>ly/s=+and+      # lyrics of all songs with 'and' in the song title
>>s/au=+t+w+s+    # songs of an author with t, w, & s in his name in that order
>> 
>> 
>>A good place to start would be sending just "Albums", which will list all album 
>>entries.  Multiple requests can be made in one letter by typing each command 
>>set on a separate line. The subject line is ignored by the server.
>> 
>>To make a request to the server, simply type the appropriate command
>>line in the body of a message to
>>    Lyrics@UMass.BITNet                                         (BITNet)
>>    Lyrics%Umass.BITNet@CUNYVM.CUNY.EDU                          (internet)
>> 
>>SUBMITTING LYRICS
>>-----------------
>> 
>>The format for Lyric Server files is fairly simple.  The title to each song 
>>must be preceded by a + which must be on the beginning of a line. The first two 
>>lines before the first song title should contain the author and album title.
>> 
>>Example:
>>Peter Gabriel
>>So
>>+Red Rain
>>--- ----
>> 
>>...(lyrics)
>> 
>>+Sledgehammer
>>------------
>> 
>>...(lyrics)
>>etc.
>> 
>>Any instrumentals contained on the album may be included for the sake of
>>completeness. On the same line as the song title should be some appropriate
>>notation indicating this.
>> 
>>Send lyrics and any other inquiries to Lyrics Manager.
>>LYRMAN%UMASS.BITNET@CUNYVM.CUNY.EDU
>> 
>>===========================================
>> 
>>IV.  Other Music-Related Mail Lists
>> 
>>    By convention, there are two addresses for every Arpa mailing list:
>> 
>>LIST@host		(the actual list)
>>LIST-request@host	(the coordinator)
>> 
>>If you wish to submit something to the list, send it to the list address.  If
>>you have an administrative request (subscriptions, problems, questions, etc.),
>>send it to LIST-request.  Those below follow this convention except where noted.
>>Rough estimates of digest frequencies are in ()'s (message forwarders have
>>relatively immediate delivery of messages after they reach the list).  Send any
>>corrections or additions to the list below to lamour%smiley@gateway.mitre.org.
>> 
>>NOTE:  Unless otherwise specified, send **ALL** administrative requests to the
>>"LIST-request" address for the list.

*****************
