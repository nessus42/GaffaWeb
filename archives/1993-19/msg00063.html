  When Ron Hill finishes his various projects (Cloudbusting, Deeper
Understanding) we could create a really wonderful CD-Rom. I believe we
could assemble a standard data format and then we would only need a
interface script for each platform. I don't know the name of the spec,
but there is one format that is supported by DOS, Mac, Amiga, UNIX and
NeXt for text files. The entire projects, and the complete archive could
easily fit on one CD-ROM, along with indexing information for each
format, with room for plenty of pictures and a few important sound
files.

>Let's talk about the text file format. I could offer TeX, PostScript, plain 
>ASCII, rtf (w/out hypertext and pictures, then it could be read by many
>MS-WINDOWS applications as MS-Word). Of course there's also the possibility
>to put it on the ftp server in several formats, but that would a) increase
>space usage and b) increase difficulty in maintaining it.

   I'll start checking out the tools used to create Windows "Help"
files. It seems to be an OK semi-hypertext format.
