In rn (and I believe a few other newsreaders) the default directory used
is ~/News.  Within that directory are a series of directories with the
hierarchical names of the newsgroup.  In the bottom level is a file
called, appropriately, KILL.  So a killfile for Love-Hounds would
reside in ~/News/rec/music/gaffa.  There are all sorts of different commands
you can put in a killfile, which are noted in the man pages to the newsreader.

If there's a subject you want to kill, just type [shift]-K whilst reading
that article.  The appropriate command will automatically be placed in the
killfile (if necessary, the software will automatically create the appropriate
directories and the actual file KILL).  The killfile is only acted upon
when you first select a newsgroup, so you'll need to quit out and then
restart that newsgroup to kill the articles.

If you want to kill a particular person's articles, for instance, me, you
can insert commands into the killfile.  Either call up the file with your
favorite editor from your shell *or* type [control]-K from within rn and
it'll automatically call up your editor with the correct killfile ready
for editing.  Try the command:  /jeffy@syrinx/h:j (on its own line).  This
will kill any article posted by me.  It won't kill articles that quote me.
